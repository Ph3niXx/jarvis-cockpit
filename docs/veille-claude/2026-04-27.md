## jarvis_applicable

### [HIGH · M] Persistent agent thread Cowork sur iOS/Android : un seul thread cross-device avec push notifs

**Source** — [Help Center Claude](https://support.claude.com/en/articles/13947068-assign-tasks-from-anywhere-in-claude-cowork)

Anthropic a déployé en beta sur Pro et Max un thread agent persistant unique, accessible depuis Claude Desktop ET Claude iOS/Android. Au lieu d'une nouvelle session par tâche, un seul thread Claude qui ne reset jamais — il garde le contexte entre tâches, on peut messager depuis le métro et continuer sur le desktop. Quand on assigne une tâche, Claude détermine le type de travail et spinne la bonne session : tâches de dev → Claude Code ; tâches knowledge → Cowork. Push notif quand une tâche est terminée ou bloquée. Rollout Max → Pro étalé sur 2 jours à partir du 24 avril. Requiert l'app Desktop ET l'app mobile installées.

> **How to apply** : architecture concurrente directe du tunnel Cloudflare actuel pour l'assistant Jarvis. Avantages d'un test pilote : pas de tunnel à maintenir, push notif natif, multi-device sans config. Limites : verrouille le contexte chez Anthropic (pas de RAG sur Supabase, pas de mémoire structurée extraite par le nightly_learner). Test : assigner une tâche "résume le brief Jarvis du jour" depuis le persistent thread mobile, comparer le résultat au mode Cloud actuel de l'onglet Jarvis. Si la qualité tient, garder le tunnel pour le RAG approfondi mais utiliser le persistent thread pour les questions courtes en mobilité.

> **Trend** : 5ᵉ produit Anthropic en 6 mois qui internalise une couche infra agent (Routines, Scheduled Tasks, Managed Agents, Memory, désormais persistent thread).

### [HIGH · S] Claude rend des charts/diagrammes interactifs inline en chat et Cowork

**Source** — [Anthropic blog](https://claude.com/blog/claude-builds-visuals) · [Help Center](https://support.claude.com/en/articles/13979539-custom-visuals-in-chat)

Lancé le 12 mars en beta, généralisé à Cowork sur tous les plans payants le 22 avril. Claude décide quand un visuel aide ou répond à des phrases-tag comme "draw this as a diagram", "chart this data". L'utilisateur peut modifier via prompts de suivi et télécharger.

> **How to apply** : direct concurrent des charts custom de l'onglet Forme et de l'onglet Musique. Pour les briefs hebdomadaires de l'onglet Brief du jour, prendre 3 mois de séances Strava + Withings et demander à Claude (Cowork) "trace l'évolution de la VMA et du poids sur la même timeline". Si la qualité tient sur du data sportif, intégrer ces visuels dans le brief comme image exportée 1×/semaine, en complément des charts qui restent live. Pas un remplaçant — un canal narratif additionnel.

> **Trend** : Anthropic absorbe la couche visualisation downstream du chat — direct concurrent des artifacts custom HTML qu'on construisait soi-même, mais dans le flow conversationnel.

### [MEDIUM · XS] Claude Code v2.1.119/120 force-rolled-back par Anthropic en 24h après 8 régressions

**Source** — [Survival checklist (gist)](https://gist.github.com/yurukusa/a866b4cd2976486156a00c190c39cef6)

Les versions v2.1.119 (23 avril) et v2.1.120 (24 avril) ont shippé 8 régressions en 24h : auto-update break, silent model swap, crashes au resume, bug UI duplication réintroduit, freeze `/mcp` sur WSL2, **CLAUDE.md ignoré**, `sandbox.excludedCommands` cassé, hang macOS sur worktree. Anthropic a force-rolled-back à v2.1.117 dans la journée. Le gist liste les workarounds par bug et pin v2.1.117 comme ancrage stable.

> **How to apply** : pour le cockpit qui dépend lourdement de Claude Code (audits hebdo, refacto specs, panel Forme récemment refacto), pin la version à v2.1.117 dans la doc d'onboarding Jarvis. Action concrète : ajouter `claude --version` en début de chaque session de travail Jarvis et logger dans un fichier work_log local. Le CLAUDE.md ignoré est le bug le plus dangereux pour MOI vu que mon CLAUDE.md fait 700+ lignes de conventions specs/archi — sans lui Claude perd les règles éditoriales et la CI lint-specs va péter en série.

> **Trend** : 3ᵉ incident Claude Code majeur en 30 jours (cache TTL → pricing flip → v120 rollback). Le rythme de release "5 releases en 6 jours" qu'Anthropic affichait fièrement la semaine dernière a clairement un coût qualité.

## claude_general

### [HIGH · S] Faille MCP "by design" : RCE sur 200k servers, Anthropic refuse de patcher

**Source** — [The Register](https://www.theregister.com/2026/04/16/anthropic_mcp_design_flaw/) · [The Hacker News](https://thehackernews.com/2026/04/anthropic-mcp-design-vulnerability.html)

OX Security publie le 16 avril une faille architecturale du MCP SDK officiel (Python, TS, Java, Rust) — RCE possible sur tout système exécutant un MCP vulnérable. Chiffres : 14 CVEs assignés, 150M+ téléchargements SDK, 7000+ MCP servers publics, jusqu'à 200k instances exposées. 9 marketplaces sur 11 ont accepté les MCP poisonnés en test. La position d'Anthropic : "le protocole fonctionne comme prévu", refus de modifier l'architecture. Permissions données aux MCP : accès direct aux user data, internal databases, API keys, chat history.

> **How to apply** : pour MOI qui utilise Cowork avec MCPs Supabase, Atlassian, Slack, Google Calendar, Gmail, Claude in Chrome — auditer Settings → Connectors aujourd'hui. Désinstaller tout MCP non-officiel ou daté de moins d'1 mois en attendant que la communauté propose des mitigations. Pour le projet cockpit qui expose ses propres tables Supabase via MCP, la responsabilité est sur Anthropic côté SDK ; rien à patcher côté projet, mais ne plus assumer "MCP officiel = sûr". Garder un œil sur red.anthropic.com pour un revirement de position.

> **Trend** : 2ᵉ écosystème Claude qui révèle une crise sécurité majeure en 6 semaines (Skills via Snyk, désormais MCP via OX) — la maturité distribution dépasse de loin la maturité sécurité.

### [MEDIUM · S] Opus 4.7 community feedback consolidé : token burn 1.5-3x, "ambiguity tax", broken output sur runs longs

**Source** — [Botmonster Tech](https://botmonster.com/posts/claude-opus-4-7-x-reddit-reception/)

Le verdict communauté 48h après la release : net-positif mais qualifié. Trois clusters de plaintes dominent. (1) Token burn — 1.5-3x plus cher en pratique vs 4.6 sur des workloads réels, au-delà du facteur tokenizer 1.46x mesuré par Simon Willison. (2) "Ambiguity tax" — le modèle ne sauve plus silencieusement les prompts vagues, demande explicitement OU appelle un tool. (3) "Impressive process, broken output" — exemple viral d'une refacto 68 minutes / multi-million tokens qui termine confidently avec une app totalement cassée. Concerns annexes : memory issues forte semaine 17 (Claude oublie des fichiers qu'il vient de créer dans la même conversation).

> **How to apply** : confirme la stratégie déjà adoptée pour le cockpit — garder Haiku 4.5 sur weekly_analysis, ne pas tester Opus 4.7 sans un budget cap doublé d'avance. Pour Jarvis mode Cloud, garder Haiku 4.5 même quand Opus 4.7 deviendra l'option par défaut côté API. Action concrète : ajouter une assertion dans le CostTracker du pipeline pour bloquer l'usage Opus 4.7 hors d'une variable d'env explicite (`ALLOW_OPUS_47=1`).

> **Trend** : Anthropic optimise sur les benchmarks (SWE-bench +6.8 points), le coût/qualité réel sur du contenu varié bouge dans l'autre sens. Discipline empirique > discipline benchmarks.

## complementary_tools

### [MEDIUM · S] Claude Agent SDK Python 0.2 : SessionStore + skills option + distributed tracing

**Source** — [GitHub Releases](https://github.com/anthropics/claude-agent-sdk-python/releases)

L'Agent SDK Python rejoint enfin le SDK TypeScript en parité features. Nouveautés : (1) `SessionStore` protocol avec 5 méthodes + helpers async store-backed pour persister/reprendre les sessions agent ; (2) top-level `skills` option dans `ClaudeAgentOptions` (`"all"` ou liste nommée) pour activer les skills sur la session principale ; (3) propagation W3C trace context du SDK au sous-process CLI quand un span OpenTelemetry est actif (corrélation SDK ↔ CLI bout en bout) ; (4) helpers `list_subagents()` / `get_subagent_messages()` pour lire les chains de messages des subagents.

> **How to install / apply** : `pip install --upgrade claude-agent-sdk` (en plus de `anthropic` qu'on utilise déjà). Cas d'usage spécifique POUR MOI : migrer le pipeline `weekly_analysis` du SDK direct (`anthropic.Anthropic` client) vers `claude-agent-sdk`. Avantage concret : si le pipeline crash en milieu de run (ex : timeout API à mi-pipeline), reprise depuis le dernier checkpoint plutôt que rerun complet → préserve la moitié du budget tokens 1$/run. Démarrer par un test isolé sur le sous-pipeline `wiki_concepts` enrichment.

> **Trend** : uniformisation cross-langage Python ↔ TS — Anthropic vise clairement Python comme langage premier de l'agent, pas juste TS pour le browser/desktop.
