## Synthèse exécutive

**Headline** — Opus 4.7 et Claude Code Routines redessinent la chaîne automation cloud, mais le tokenizer change rend le coût réel 20-30% plus élevé.

**3 tendances de fond**

1. Anthropic centralise l'automation côté cloud (Routines, Scheduled Tasks desktop, `/loop` CLI) — plus besoin de GitHub Actions pour faire tourner un agent récurrent.
2. Régressions silencieuses qui touchent le portefeuille (cache TTL passé à 5min, tokenizer 4.7 +35% de tokens) — la discipline de monitoring des coûts devient un quick-win plus rentable que l'optimisation de prompt.
3. L'écosystème skills tiers mûrit (`evals-skills` de Hamel Husain, marketplace Cowork formel) — ce qu'on écrit en skill maison est de plus en plus distribuable.

**Quick win cette semaine** — Forcer `cache_control.ttl=3600s` dans le pipeline `weekly_analysis` pour neutraliser la régression du cache 5min Claude Code (S effort, high impact).

**À surveiller** — La pricing confusion Claude Code Pro→Max ($20 → $100 puis revert en 24h) suggère qu'Anthropic teste activement la monétisation premium. Probable que le cap Pro se durcisse dans les 3 mois.

### [HIGH · S] Claude Opus 4.7 livré le 16 avril : tokenizer change, +13% coding, prix stable

**Source** — [Anthropic Docs](https://platform.claude.com/docs/en/about-claude/models/whats-new-claude-4-7)

Opus 4.7 GA le 16 avril 2026. SWE-bench Verified passe de 80.8% à 87.6%, SWE-bench Pro 53.4 → 64.3%. Prix identique à 4.6 ($5/$25 par 1M tokens) **mais** nouveau tokenizer qui produit jusqu'à 35% de tokens en plus pour le même texte → hausse silencieuse 20-30% du coût réel. Nouveau niveau d'effort `xhigh` pour les tâches lourdes.

> **How to apply** : sur le pipeline hebdo Claude Haiku, valider que le CostTracker (budget 1$/run) ne saute pas en silence avec le nouveau tokenizer si on bascule un jour sur 4.7. Re-mesurer empiriquement input_tokens vs longueur de prompt sur un run pilote avant de promouvoir Opus 4.7 pour des analyses stratégiques. Garder Haiku pour le volume.

> **Trend** : cinquième mise à jour mineure d'Opus en 6 mois — cadence stable, pas de rupture API.

### [HIGH · M] Claude Code Routines : automation cloud planifiée (research preview)

**Source** — [9to5Mac / Anthropic](https://9to5mac.com/2026/04/14/anthropic-adds-repeatable-routines-feature-to-claude-code-heres-how-it-works/)

Routines = configuration Claude Code (prompt + repos + connecteurs) sauvegardée et exécutée automatiquement sur cron / API / event GitHub. Tourne sur l'infra Anthropic, pas besoin que ton Mac soit allumé. Pro = 5/jour, Max = 15/jour, Team = 25/jour. Lancé le 14 avril en research preview.

> **How to apply** : concurrent direct des GitHub Actions cron qui exécutent les pipelines quotidiens et hebdo. Pour la veille IA et l'enrichissement Wiki, une Routine pourrait remplacer un workflow YAML + secrets, et garder la même prose Claude. Tester en double-run sur la veille Claude pendant 2 semaines avant de migrer le cron principal.

> **Trend** : Anthropic continue de pousser le scheduling natif (Scheduled Tasks desktop, `/loop` CLI, maintenant Routines cloud) — convergence claire vers l'automation managée.

### [HIGH · S] Régression silencieuse : cache TTL Claude Code passé de 1h à 5min

**Source** — [XDA Developers](https://www.xda-developers.com/anthropic-quietly-nerfed-claude-code-hour-cache-token-budget/)

Anthropic a fait passer le TTL par défaut du prompt cache de 1h à 5min (probablement dès début mars, confirmé en avril). Conséquence : sessions longues qui s'arrêtent >5min payent un `cache_creation` au tarif d'écriture (5x le tarif lecture). Utilisateurs documentés : x5 le coût quotidien Claude Code, jusqu'à 60% de waste sur certains workloads.

> **How to apply** : pour le pipeline `weekly_analysis` qui chaîne plusieurs prompts Claude Haiku, vérifier si on bénéficie réellement du cache (logs API). Si oui, passer `cache_control.ttl=3600` explicitement pour forcer 1h. Côté Cowork (cette tâche planifiée), watcher la facture mensuelle — un run de veille de 10min peut désormais payer le cache 2x.

> **Trend** : quatrième fois en 4 mois qu'une régression silencieuse Claude Code remonte — discipline de monitoring des coûts à durcir.

### [MEDIUM · S] Subagents Claude Code en worktree isolé : bug fix + pattern parallélisé

**Source** — [claudefa.st](https://claudefa.st/blog/guide/development/worktree-guide)

`isolation: worktree` dans le frontmatter d'un subagent → le subagent tourne dans un git worktree temporaire. Si pas de change, cleanup auto. Bug récent fix (avril) : les subagents en worktree pouvaient se voir refuser Read/Edit dans leur propre worktree. Permet de lancer 2 subagents qui éditent le même fichier en parallèle (2 approches différentes) puis comparer.

> **How to apply** : pour la routine hebdomadaire d'audit / spec-drift-check : déclencher 2 subagents en worktree pour produire 2 versions concurrentes d'une refacto (ex : refacto du data layer Tier 2 du cockpit). Le main agent compare les diffs et pick. Évite le risque de partial edit qui casse le mount React entre deux étapes.

### [HIGH · M] evals-skills (Hamel Husain) : 6 skills pour structurer les évaluations LLM

**Source** — [Hamel Husain (GitHub)](https://github.com/hamelsmu/evals-skills)

Plugin Claude Code open source qui apporte 6 skills d'évaluation : `error-analysis`, `generate-synthetic-data`, `write-judge-prompt`, `validate-evaluator`, `evaluate-rag`, `build-review-interface`. Issu de l'expérience de Hamel sur 50+ entreprises et 4000+ étudiants. Complète les MCP servers d'observabilité (qui exposent les traces) en apprenant à l'agent quoi en faire.

> **How to install / apply** : installation via `claude plugin add hamelsmu/evals-skills`. Premier essai : lancer `evaluate-rag` sur le RAG Jarvis (le retriever sémantique au-dessus de la table vectorielle). Objectif : produire un jeu d'éval (paires question / chunk attendu) et une note de précision retrieval. Skill `error-analysis` utile pour comprendre pourquoi certaines réponses Jarvis ratent malgré chunks corrects.

> **Trend** : l'écosystème skills tiers commence à mûrir au-delà des skills Anthropic — premiers signaux d'un vrai marketplace open.

### [MEDIUM · XS] Diff system prompt Opus 4.6 → 4.7 : moins de relances, tools-first

**Source** — [Simon Willison](https://simonwillison.net/2026/Apr/18/opus-system-prompt/)

Le system prompt d'Opus 4.7 retire les vieilles consignes politiques périmées et change deux comportements clés : (1) le modèle ne relance plus quand l'user veut quitter la conv, (2) face à une demande ambiguë il tente une réponse raisonnable plutôt que d'enquêter, et appelle un outil pour lever l'ambiguïté avant de poser une question.

> **How to apply** : si tu as des instructions custom dans CLAUDE.md ou des skills qui forcent Claude à "demander avant d'agir" ou "demander des précisions", elles vont jurer avec le nouveau comportement par défaut. Relire les system prompts d'agents et virer les instructions redondantes — Claude le fait déjà.

> **Trend** : Anthropic affine ses prompts de production toutes les 6-8 semaines — checkpoint utile pour ré-aligner les CLAUDE.md projet.
